Cowrie — Honeypot (Panduan Ringkas)

Materi tambahan telah dirapikan: fitur utama, instalasi, konfigurasi, penggunaan, logging, dan troubleshooting untuk lab terisolasi.

Overview & Fitur

  • SSH/Telnet Honeypot: emulasi layanan dan interaksi penyerang.
  • Command Logging & Session Recording.
  • Fake File System & File Download Capture.

Instalasi (Ringkas)

# Clone versi stabil
git clone -b v2.4.0 https://github.com/cowrie/cowrie
cd cowrie

# Virtual environment (Linux/macOS)
python3 -m venv cowrie-env
source cowrie-env/bin/activate

# Virtual environment (Windows PowerShell)
python -m venv cowrie-env
.\cowrie-env\Scripts\Activate.ps1

# Dependencies
pip install --upgrade pip
pip install -r requirements.txt
pip install -e .

Konfigurasi Dasar

[honeypot]
hostname = server
listen_endpoints = tcp:2222:interface=0.0.0.0

[output_json]
enabled = true

Opsional: port-forwarding ke 22 pada host (Linux) dengan iptables di lab terisolasi.

Penggunaan

# Start/Stop/Status
cowrie start
cowrie stop
cowrie status

Anda dapat membuat skrip/alias untuk restart cepat dan melihat log real-time.

Lokasi Log & Analisis

  • Main Log: var/log/cowrie/cowrie.log; JSON: var/log/cowrie/cowrie.json
  • Downloads: var/lib/cowrie/downloads/; TTY: var/lib/cowrie/tty/
# Contoh analisis dengan jq (Linux)
cat var/log/cowrie/cowrie.json | jq 'select(.eventid=="cowrie.login.success")'
cat var/log/cowrie/cowrie.json | jq 'select(.eventid=="cowrie.command.input")'

Troubleshooting (Umum)

  • Port in use: periksa dengan netstat dan hentikan proses terkait.
  • Permission: jalankan sebagai non-root; cek izin file environment.
  • Virtualenv hilang: buat ulang venv dan install ulang dependencies.

Best Practices Keamanan

  • Jalankan di segment jaringan terisolasi; jangan di produksi.
  • Monitor dan backup log secara berkala.
  • Batasi akses dan amankan storage log.

Cowrie Honeypot — Monitoring Interactions

Cowrie adalah honeypot SSH/telnet yang mengemulasikan layanan untuk mengumpulkan taksonomi serangan. Fokus halaman ini pada deployment yang aman, logging, dan integrasi analytic.

Deployment Principles

  • Letakkan honeypot pada segment jaringan terisolasi; jangan gunakan di jaringan produksi tanpa mitigasi.
  • Pastikan logging diarahkan ke storage terpisah dan dilindungi untuk analisis forensik.
