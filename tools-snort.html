Snort — Rule & Penjelasan

Contoh rule yang digunakan untuk monitoring di lab serta tips tuning untuk mengurangi false positive.

Contoh Rule

alert tcp any any -> 192.168.56.101 80 (msg:"POTENTIAL SQL KEYWORD in HTTP"; content:"select"; nocase; http_client_body; sid:1000001; rev:1;)

Aturan ini bersifat heuristik. Lakukan tuning (threshold, whitelist) dan uji di lab sebelum disebarkan ke produksi.

Ringkasan dari Arsip (Non-aksi)

  • Snort digunakan untuk analisis trafik real-time dan logging paket.
  • Struktur konfigurasi umum: snort.lua dan direktori rules/ untuk local.rules.
  • Untuk lab, jalankan mode monitor dan evaluasi alert, hindari konfigurasi blocking di jaringan produksi.

Snort IDS — Intrusion Detection System

Snort adalah IDS berbasis signature/port analysis. Halaman ini fokus pada rule design, tuning, dan integrasi dengan pipeline monitoring.

Rule Engineering

Desain rule yang efektif memerlukan pemahaman traffic baseline, false-positive management, dan threat modeling. Gunakan alert grouping dan thresholding untuk menurunkan noise.

Integrasi

  • Sambungkan Snort ke SIEM untuk korelasi event.
  • Gunakan pcap capture untuk verifikasi true-positive.
