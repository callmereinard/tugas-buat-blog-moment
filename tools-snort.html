Snort — Rule Contoh & Running di Lab

Konten tambahan dirapikan: sintaks dasar, contoh rule deteksi, dan cara menjalankan Snort di lingkungan lab.

Sintaks Dasar

alert [protocol] [src_ip] [src_port] -> [dst_ip] [dst_port] (rule options)

Contoh Rule Deteksi

# Deteksi Nmap SYN Scan
alert tcp any any -> any any (msg:"Nmap SYN Scan Detected"; flags:S; sid:1000001; rev:1;)

# Deteksi ICMP Flood
alert icmp any any -> any any (msg:"HPING3 ICMP Flood Detected"; sid:1000004; rev:1;)

# Heuristik kata kunci SQL di HTTP body (lab-only)
alert tcp any any -> 192.168.56.101 80 (msg:"Potential SQL keyword"; content:"select"; nocase; http_client_body; sid:1000010; rev:1;)

Lakukan tuning (threshold, whitelist) dan uji di lab sebelum dipakai luas untuk mengurangi false positive.

Menjalankan Snort (Lab)

sudo snort -c /etc/snort/snort.lua -R /etc/snort/rules/local.rules -i eth0 -A alert_fast

Gunakan mode alert_fast untuk melihat ringkas alert; pastikan interface dan path sesuai lingkungan Anda.

Ringkasan

  • Konfigurasi umum: snort.lua dan rules/local.rules.
  • Gunakan lab terisolasi untuk evaluasi alert dan tuning aturan.

Snort IDS — Intrusion Detection System

Snort adalah IDS berbasis signature/port analysis. Halaman ini fokus pada rule design, tuning, dan integrasi dengan pipeline monitoring.

Rule Engineering

Desain rule yang efektif memerlukan pemahaman traffic baseline, false-positive management, dan threat modeling. Gunakan alert grouping dan thresholding untuk menurunkan noise.

Integrasi

  • Sambungkan Snort ke SIEM untuk korelasi event.
  • Gunakan pcap capture untuk verifikasi true-positive.
