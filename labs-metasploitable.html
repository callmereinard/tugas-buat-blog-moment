Metasploitable 2 — Studi Kasus (Lab Aman)

Konten tambahan telah disusun ulang: hanya untuk lingkungan terisolasi. Fokus pada setup, observabilitas, dan pemahaman layanan rentan, tanpa instruksi eksploitasi.

Informasi Lab

  • VM: Metasploitable 2 (Ubuntu 8.04)
  • Tujuan: Image sengaja rentan untuk pelatihan keamanan
  • Kesulitan: Pemula

Setup & Instalasi

  • Unduh dari SourceForge: Metasploitable 2
  • Ekstrak arsip dan impor ke VirtualBox: File → Import Appliance

Enumerasi & Reconnaissance

Lakukan pemindaian servis untuk memetakan permukaan serangan (lab-only):

# Contoh pemindaian versi layanan
nmap -sV -p- <IP_Metasploitable>

# Cuplikan output umum (ilustrasi)
PORT     STATE SERVICE     VERSION
21/tcp   open  ftp         vsftpd 2.3.4
22/tcp   open  ssh         OpenSSH 4.7p1
23/tcp   open  telnet      Linux telnetd
80/tcp   open  http        Apache httpd
139/tcp  open  netbios-ssn Samba smbd 3.X
445/tcp  open  netbios-ssn Samba smbd 3.X
3306/tcp open  mysql       MySQL

Gunakan hasil enumerasi untuk memahami layanan yang lemah dan menyiapkan deteksi/mitigasi.

Layanan Penting & Catatan Keamanan

  • FTP vsftpd 2.3.4: versi historis dengan isu keamanan terdokumentasi.
  • Samba: periksa konfigurasi dan praktik kredensial.
  • UnrealIRCd/Tomcat: layanan default dan hardening diperlukan.
  • MySQL: lakukan audit kebijakan password dan hak akses.
  • Aplikasi Web (Mutillidae, DVWA): gunakan untuk belajar deteksi dan mitigasi (lab-only).

Observabilitas & Hardening

  • Kumpulkan pcap, log aplikasi, dan snapshot VM sebelum/selesai eksperimen.
  • Tambah IDS rules untuk pola scan dan anomali autentikasi.
  • Terapkan least privilege dan patching pada layanan yang diuji.

Checklist Lab

  • [x] Pemindaian jaringan dasar dengan Nmap
  • [ ] Review layanan utama (FTP, Samba, IRC, HTTP, DB)
  • [ ] Uji deteksi IDS untuk skenario umum
  • [ ] Dokumentasi temuan dan rekomendasi mitigasi
Pengingat: Metasploitable 2 sangat rentan. Jangan pernah expose ke internet. Gunakan hanya pada jaringan terisolasi.

Metasploitable 2 — Studi Kasus

Analisis layanan rentan pada image Metasploitable 2 sebagai lingkungan latihan. Fokus pada observabilitas, log analysis, dan mitigasi.

Tujuan Pembelajaran

  • Memahami layanan umum yang rentan (mis. versi lama layanan web, database).
  • Menggunakan logging dan network capture untuk mengidentifikasi tanda-tanda exploitation.
  • Menyusun rekomendasi mitigasi berbasis konfigurasi dan patching.

Catatan Riset (ringkasan teknis)

Jalankan image hanya pada jaringan terisolasi. Gunakan pcap, proses list, dan snapshot VM untuk mengumpulkan bukti. Fokus analisis pada: diskrepansi versi paket, endpoint terbuka, dan konfigurasi default yang lemah.
