Wazuh — SIEM & Monitoring (Ringkas)

Materi tambahan disusun ulang untuk lab: arsitektur Wazuh, langkah instalasi inti, konfigurasi, dan praktik aman.

Arsitektur

  • Wazuh Manager: menerima dan menganalisis data dari agent.
  • Wazuh Agent: dikinstal di endpoint untuk mengirim log.
  • Elastic Stack: Elasticsearch + Kibana untuk indexing dan visualisasi.

Instalasi Inti (Debian/Ubuntu)

# Tambah repository Wazuh (ringkas)
echo "deb [signed-by=/usr/share/keyrings/wazuh.gpg] https://packages.wazuh.com/4.x/apt/ stable main" | \
sudo tee -a /etc/apt/sources.list.d/wazuh.list

sudo apt update
sudo apt install wazuh-manager filebeat -y

# Elasticsearch & Kibana (versi sesuai lingkungan)
sudo apt install elasticsearch -y
sudo systemctl enable --now elasticsearch

# Install Kibana dan plugin Wazuh (contoh versi)
sudo dpkg -i kibana-7.17.13-amd64.deb
sudo -u kibana /usr/share/kibana/bin/kibana-plugin install file:///path/to/wazuh_kibana.zip

Sesuaikan versi dan repository sesuai dokumentasi resmi Wazuh/Elastic.

Konfigurasi & Agen

  • Konfigurasi dasar di /var/ossec/etc/ossec.conf (output JSON, syslog, rules).
  • Registrasi agent via manage_agents dan wazuh-agent pada endpoint.
  • Gunakan Filebeat template dari Wazuh untuk pipeline Elastic.

Operasional & Troubleshooting

# Service (contoh)
sudo systemctl status wazuh-manager
sudo systemctl status elasticsearch
sudo systemctl status kibana

# Log
sudo tail -f /var/ossec/logs/ossec.log
sudo journalctl -u wazuh-manager -f
  • Pastikan memori heap Elasticsearch disetel sesuai host.
  • Verifikasi konektivitas agent (port 1514) dan firewall.

Praktik Aman

  • Ubah kredensial default; aktifkan HTTPS untuk Kibana.
  • Restriksi akses dashboard; backup konfigurasi dan indeks.
  • Integrasi Slack/VirusTotal hanya di lab dan amankan API keys.
