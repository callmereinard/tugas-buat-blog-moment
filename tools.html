Prinsip Penggunaan Aman

  • Jalankan tools hanya di lingkungan terotorisasi/terisolasi.
  • Kombinasikan observability (pcap, logs) untuk analisis pasca-aktivitas.
  • Dokumentasikan konfigurasi, rule sets, dan perubahan untuk reproducibility dan audit.

Contoh Penggunaan Alat Secara Aman

Nmap — discovery & service enumeration

Contoh perintah yang umum dipakai untuk enumerasi di lab terkontrol:

# pindai port umum dan detect service
nmap -p 1-65535 -sV --version-intensity 0 192.168.56.101

Gunakan output untuk merencanakan observasi lebih lanjut dan pencatatan versi vulnerable components.

Wireshark/Tshark — capture dan filter

Contoh display filter untuk melihat hanya traffic HTTP:

http && ip.addr == 192.168.56.101
Burp Suite (Community) — Testing aplikasi web di lab

Konfigurasi singkat: atur browser untuk menggunakan Burp sebagai proxy, hanya aktifkan intercept untuk request ke host lab yang diizinkan, dan catat semua request/responses untuk analisis.

Snort — rule contoh deteksi pola (IDS)

Contoh rule sederhana untuk mendeteksi pola query yang mengandung kata kunci SQL umum (untuk tujuan monitoring di lingkungan lab):

alert tcp any any -> 192.168.56.101 80 (msg:"POTENTIAL SQL KEYWORD in HTTP"; content:"select"; nocase; http_client_body; sid:1000001; rev:1;)

Catatan: rule di atas bersifat heuristik dan bisa menimbulkan false positives; gunakan bersama aturan lain dan tuning kontekstual.

Wazuh / SIEM — korelasi dan alerting

Gunakan Wazuh untuk mengumpulkan log host (syslog, application logs) dan korelasikan event dari IDS untuk membangun workflow deteksi dan respons.
